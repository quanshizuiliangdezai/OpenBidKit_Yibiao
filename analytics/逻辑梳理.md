# Analytics 逻辑梳理

## 核心原则

| 项 | 核心逻辑 |
| --- | --- |
| AE | 保持现有埋点采集字段不变，保存实时/短期明细 |
| ANALYTICS_DB | `openbidkit-analytics` 只保存简化后的新版 `stats_*` 长期统计表 |
| RESOURCE_DB | `openbidkit-resources` 保存资源元数据和历史累计点击量 |
| Queue | 不使用 Queue，不保留 Queue 代码 |
| `/track` | 写 Analytics Engine；仅对 `client_created_at` 距当前业务日期不超过 1 天的新客户端尝试实时写入 D1 |
| Cron | 北京时间每天 02:00 汇总前一天 AE 数据到 D1 |
| 历史范围 | 直接读 D1，忽略 AE |
| 今天/7天/30天 | 直接读 AE，忽略 D1 |
| 最近事件/留存/客户端详情 | 只读 AE，不持久化 |
| 公告管理 | 不属于埋点统计改造，保持不变 |

## 存储与绑定

| 绑定 | 类型 | 用途 |
| --- | --- | --- |
| `ANALYTICS` | Analytics Engine dataset `agnet_analytics` | 详细埋点事件、实时查询、Cron 汇总来源 |
| `ANALYTICS_DB` | D1 `openbidkit-analytics` | 长期埋点统计新版 `stats_*` 表 |
| `RESOURCE_DB` | D1 `openbidkit-resources` | 资源元数据和累计点击量 |
| `RESOURCE_BUCKET` | R2 `openbidkit` | 资源图片 |
| `NOTICE_STORE` | KV | 公告、GitHub stats 缓存 |

## 事件类型

| event | 统计用途 |
| --- | --- |
| `app_open` | 打开次数、留存 |
| `page_view` | 页面访问量、页面排行 |
| `config_usage` | 配置使用分布 |
| `ai_request` | 模型使用、累计 AI 请求 |
| `resource_click` | 资源点击统计 |

## ANALYTICS_DB 新版表

| 表 | 主键 | 关键字段 | 查询用途 |
| --- | --- | --- | --- |
| `stats_totals` | `project_name` | `total_clients`, `total_open`, `total_page_views`, `total_events`, `total_ai_requests`, `last_rollup_date` | 概览累计总数 |
| `stats_clients` | `project_name + client_id` | `first_seen_at`, `first_seen_date`, `active_days`, `last_active_date`, `last_active_version` | 今日/7日新增、客户端统计列表 |
| `stats_daily` | `project_name + activity_date` | `active_clients`, `app_open_count`, `page_view_count`, `event_count`, `ai_request_count` | 概览每日统计历史部分 |
| `stats_pages` | `project_name + page` | `view_count` | 访问分析历史页面排行 |
| `stats_versions` | `project_name + version` | `event_count`, `client_count` | 访问分析历史版本分布 |
| `stats_configs` | `project_name + field_key + value` | `report_count` | 配置使用历史分布 |
| `stats_models` | `project_name + request_type + provider + endpoint_host + model` | `request_count`, `total_tokens` | 模型使用历史分布、Token 和筛选 |
| `stats_rollup_runs` | `project_name + activity_date` | `status`, `started_at`, `completed_at`, `error` | 防止同一天重复汇总 |

## Cron 写入逻辑

| 步骤 | 查询/操作 | 写入表 |
| --- | --- | --- |
| 业务日期 | `activity_date = 昨日 Asia/Shanghai`，Cron 为 `0 18 * * *` | 无 |
| 项目发现 | AE 查询昨日 `blob1` 分组 | 无 |
| 重复保护 | `stats_rollup_runs.status='success'` 则跳过 | `stats_rollup_runs` |
| 总量 | AE 统计昨日 `app_open/page_view/ai_request/allowed events` | `stats_totals`, `stats_daily` |
| 活跃客户端 | AE 昨日任意允许事件 `COUNT(DISTINCT blob7)` | `stats_daily.active_clients` |
| 客户端生命周期 | AE 昨日按 `blob7` 分组，访问天数加 1，更新最近活跃日期和版本 | `stats_clients` |
| 页面 | AE `event='page_view'` 按 `blob3` 分组累加访问量 | `stats_pages` |
| 版本 | AE 按空版本归 `未知版本` 分组累加事件量；客户端数从 `stats_clients.last_active_version` 分组重算覆盖 | `stats_versions` |
| 配置 | AE `event='config_usage'` 按配置字段分组累加次数 | `stats_configs` |
| 模型 | AE `event='ai_request'` 按 `blob12/blob9/blob10/blob11` 分组累加请求次数和 `double4` Total Tokens | `stats_models` |
| 资源 | AE `event='resource_click'` 按 `blob9` 分组，把点击量累加到资源表 | `RESOURCE_DB.resources.click_count` |

## 查询逻辑

### 概览

| 指标 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 总客户端数 | D1 | `stats_totals.total_clients` |
| 总打开次数 | D1 | `stats_totals.total_open` |
| 总页面访问量 | D1 | `stats_totals.total_page_views` |
| 总事件数 | D1 | `stats_totals.total_events` |
| 累计 AI 请求数 | D1 | `stats_totals.total_ai_requests` |
| 今日新增客户端 | D1 | `stats_clients.first_seen_date = 今日 Asia/Shanghai` |
| 7日新增客户端 | D1 | `stats_clients.first_seen_date >= 今日往前6天` |
| 今日活跃客户端 | AE | 今日任意允许事件 `COUNT(DISTINCT blob7)` |
| 每日统计 | D1 + AE | 今天从 AE 查；前 9 天从 `stats_daily` 查 |
| 留存概览 | AE | 保持 `/api/retention` 逻辑，不持久化 |
| GitHub stats | GitHub + KV | 保持现有缓存逻辑 |

### 客户端统计

| 视图 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 客户端列表 | D1 | `stats_clients` 全量，按最近活跃日期和首次访问倒排 |
| 客户端详情 | AE | 按 `projectName + client_id` 查询 7天/30天/全部，按日期和事件分组 |

### 访问、配置、模型

| 模块 | 历史 | 今天/7天/30天 |
| --- | --- | --- |
| 访问分析 | D1 `stats_pages` / `stats_versions` | AE 实时聚合 |
| 配置使用 | D1 `stats_configs` | AE 实时聚合 |
| 模型使用 | D1 `stats_models` | AE 实时聚合 |

版本排序由 Worker JS 做语义版本倒排，保证 `2.10.1 > 2.9.1`，`未知版本` 放最后。历史版本客户端数来自 `stats_clients.last_active_version` 当前分组，不按天累加。

模型使用同时返回请求次数和 Total Tokens。历史 Total Tokens 来自 `stats_models.total_tokens`；今天/7天/30天直接从 AE `double4 * _sample_interval` 聚合。

### 最近事件

| 指标 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 总数 | AE | `COUNT()`，可按事件过滤 |
| 列表 | AE | `ORDER BY timestamp DESC, blob7 DESC, blob2 DESC, blob3 DESC LIMIT 10 OFFSET pageOffset` |
| 事件筛选 | AE | `blob2 = event`，只允许白名单事件 |

### 资源管理

| 功能 | 数据源 | 查询逻辑 |
| --- | --- | --- |
| 资源公开列表 | `RESOURCE_DB` + AE | `resources.enabled=1`，点击量为 `resources.click_count + AE 今天点击量` |
| 资源后台列表 | `RESOURCE_DB` + AE | `resources` 全量，点击量为 `resources.click_count + AE 今天点击量` |
| 资源图片 | `RESOURCE_BUCKET` | `/resource-image` 读取 R2 |
| 资源历史点击 | `RESOURCE_DB` | Cron 每天累加到 `resources.click_count` |

## 历史回填

历史回填脚本复用 `rollupStatsDay()` 写 `ANALYTICS_DB` 的统计表，但资源点击不按天累加。回填结束时按 AE 中今天之前的总资源点击量直接设置 `RESOURCE_DB.resources.click_count`，避免脚本重跑导致资源累计量翻倍。

新增字段补齐脚本只处理 `stats_versions.client_count` 和 `stats_models.total_tokens`：版本客户端数从 `stats_clients.last_active_version` 分组覆盖，模型 Total Tokens 从 AE 历史数据覆盖写入，不重跑每日统计，不处理资源点击。

## 接口与数据源

| 接口 | 数据源 | 核心逻辑 |
| --- | --- | --- |
| `POST /track` | AE + D1 | 写 AE，实时 upsert 近 1 天新客户端 |
| `GET /api/overview` | D1 + AE + KV | 概览总数、新增、今日活跃、每日统计 |
| `GET /api/clients` | D1 | 客户端列表 |
| `GET /api/client-detail` | AE | 单客户端日期和事件明细 |
| `GET /api/traffic` | D1 或 AE | 历史读 D1；今天/7天/30天读 AE |
| `GET /api/config-usage` | D1 或 AE | 历史读 D1；今天/7天/30天读 AE |
| `GET /api/model-usage` | D1 或 AE | 同上，支持服务商/域名/模型筛选 |
| `GET /api/latest` | AE | 最近事件分页和事件筛选 |
| `GET /api/retention` | AE | 留存概览，不持久化 |
| `GET /api/projects` | D1 优先，AE 兜底 | 项目名列表 |
| `GET/POST/DELETE /api/notice` | KV | 公告后台管理 |
| `GET/POST/DELETE /api/resources` | `RESOURCE_DB` + R2 + AE | 资源管理 |
| `GET /api/github-repo-stats` | GitHub + KV | 仓库统计和缓存 |
